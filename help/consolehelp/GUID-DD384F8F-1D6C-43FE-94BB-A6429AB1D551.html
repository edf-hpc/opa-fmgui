opashowmc

(Linux) Displays the Intel® Omni-Path Multicast groups created for the fabric along with the Intel® Omni-Path Host Fabric Interface (HFI) ports which are a member of each multicast group. This command can be helpful when attempting to analyze or debug Intel® Omni-Path multicast usage by applications or ULPs such as IPoIB.

Syntax

opashowmc [-v] [-t portsfile] [-p ports]

or 

opashowmc --help

Options

--help

Produces full help text.

-v

Returns verbose output and shows name of each member.

-t portsfile

File with list of local HFI ports used to access fabric(s) for analysis. Default is /etc/sysconfig/opa/ports file.

-p ports

List of local HFI ports used to access fabric(s) for analysis. Default is first active port. The first HFI in the system is 1. The first port on an HFI is 1.

Uses the format hfi:port, for example:

0:0

First active port in system.

0:y

Port y within system.

x:0

First active port on HFI x.

x:y

HFI x, port y.

Examples 

opashowmc
opashowmc -p '1:1'
opashowmc -p '1:1 2:1'

Environment Variables

The following environment variables are also used by this command:

PORTS

List of ports, used in absence of -t and -p.

PORTS_FILE

File containing list of ports, used in absence of -t and -p.
